# Hive 离线数仓开发 Skill

## 概述

本 Skill 用于指导 Hive 离线数据仓库的开发工作，涵盖从公共数仓层（DWD/DWM）到应用数仓层（DM/APP）的完整开发规范与最佳实践。

## 适用场景

- 新建 Hive 数仓表（DWD / DWM / DM / APP 各层）
- 编写 HiveSQL ETL 脚本
- 数仓分层建模与表命名规范咨询
- 数据质量校验与监控方案设计
- 数仓性能调优

## 使用指南

### 1. 阅读开发规范

在开始数仓开发之前，**必须先阅读**对应层级的开发规范文档：

| 层级 | 参考文档 | 说明 |
|------|----------|------|
| DWD / DWM | [dwd_dwm_development_rules.md](references/dwd_dwm_development_rules.md) | 公共数仓层开发规则 |
| DM / APP | [dm_app_development_rules.md](references/dm_app_development_rules.md) | 应用数仓层开发规则 |

### 2. 运行辅助脚本

本 Skill 提供了辅助 Python 脚本，可在开始开发前执行以确认环境就绪：

```bash
python scripts/start.py
```

```
┌───────────────────────────────────────────────────┐

## 规范要点速查

### 命名规范
- **库名**: `{层级}_{业务域}` — 例如 `dwd_trade`, `dm_marketing`
- **表名**: `{层级}_{业务域}_{数据域}_{描述}_{更新周期}` — 例如 `dwd_trade_order_detail_di`
- **字段名**: 使用小写下划线命名，有含义的英文缩写

### 更新周期标识
| 标识 | 含义 |
|------|------|
| `di` | 每日增量 (daily incremental) |
| `df` | 每日全量 (daily full) |
| `hi` | 每小时增量 (hourly incremental) |
| `mi` | 每月增量 (monthly incremental) |

### 分区策略
- 时间分区: `dt` (日期分区, 格式 `YYYYMMDD`)
- 小时分区: `hour` (格式 `HH`)
- 业务分区: 根据业务需要添加
